# Prompt F. README, встроенная справка, welcome и продуктовый copywriting

## Для кого

Этот prompt предназначен для отдельного агента GPT-5.4 с максимальным effort. Агент отвечает не за код runtime, а за человеческий слой продукта: README, help, welcome, подсказки и словарь типовых ошибок.

## Обязательный контекст перед стартом

Прочитай:

1. `docs/client-mode-ru-rollout/00-master-spec.md`
2. `docs/client-mode-ru-rollout/01-parallel-workstreams.md`
3. текущий `README.md`
4. текущие welcome/help-related строки в `app/ui/dialogs.py`, `app/ui/main_window.py`, `app/ui/settings.py`

## Твоя зона владения

Можно свободно менять:

- `README.md`
- `README.en.md`, если создаёшь companion document
- `app/help/*`
- дополнительные markdown/content files для встроенной справки
- отдельные словари/глоссарии типовых ошибок

Можно точечно менять:

- `docs/client-mode-ru-rollout/*`, если нужно добавить контентную карту или glossary reference

Не трогай без крайней необходимости:

- `app/runtime/*`
- core UI-виджеты
- build scripts

## Главная цель

Полностью переписать пользовательские тексты так, чтобы продукт был понятен человеку без техподготовки.

Приоритеты:

- понятность
- короткие формулировки
- последовательный тон
- русский как основной язык
- смысловая синхронность с английским вариантом

## Жёсткие требования к тексту

Русские тексты должны быть:

- литературными
- короткими
- дружелюбными
- спокойными
- без машинных кальк
- без жаргона, если можно объяснить проще

Стиль:

- одно предложение = одна мысль
- сначала понятный вывод, потом техническая деталь
- если технический термин неизбежен, рядом даётся простое объяснение

Примеры направления:

- не `TCP Reachability failed`, а `Сервер не отвечает`
- ниже пояснение: `Проверка соединения по TCP не удалась`

- не `Primary route owner`, а `Сейчас через это подключение идёт интернет`
- в техблоке допустимо `Основной маршрут`

- не `Handshake stale`, а `Данные о соединении устарели`
- пояснение: `Нужно проверить подключение заново`

## Что именно нужно реализовать

### 1. Новый русский `README.md`

Полностью перепиши `README.md` под обычного пользователя.

Обязательные разделы:

- `Что это`
- `Как запустить`
- `Как добавить подключение`
- `Как подключиться`
- `Если что-то не работает`
- `Какой файл скачать для Mac`

README должен объяснять:

- что это за программа
- как её запустить
- как добавить конфиг
- как подключиться
- как сделать подключение основным
- как отключиться
- как понять, что всё работает
- что делать при ошибках
- как выбрать архив Windows/macOS/Apple Silicon/Intel

### 2. Companion English doc

Если проект всё ещё ориентирован и на англоязычного пользователя, создай `README.en.md` с тем же смыслом. Он не должен быть “хуже” русского по качеству.

### 3. Встроенная справка

Подготовь контент для встроенного окна `Справка`.

Обязательные разделы:

- `С чего начать`
- `Как работает подключение`
- `Что значит “Сделать основным”`
- `Что показывают статусы`
- `Что такое TCP-проверка`
- `Что делать, если соединение не запускается`
- `Как понять, что у меня Mac на Apple Silicon или Intel`

Нужно подготовить версии:

- `ru`
- `en`

Рекомендуемый формат:

- markdown-файлы внутри `app/help/`
- или другой простой ресурсный формат, который UI-поток сможет легко подгрузить

### 4. Welcome / quick start copy

Подготовь новый сценарий welcome-flow:

1. Добавьте своё подключение.
2. Выберите его в списке.
3. Нажмите `Подключить`.
4. Если нужно, нажмите `Сделать основным`.
5. Проверьте состояние в правой панели.

Нужны:

- краткая версия для quick start
- более подробная версия для help
- английский аналог

### 5. Словарь типовых ошибок

Создай компактный человеческий словарь типовых проблем и объяснений.

Минимальный набор:

- сервер не отвечает
- не удалось запустить движок
- порт уже занят
- данные авторизации не подошли
- системный proxy не удалось применить
- WireGuard требует подтверждения
- данные о соединении устарели

Для каждого кейса подготовь:

- короткий заголовок
- простое объяснение
- что сделать дальше
- английский аналог

Этот словарь должен быть пригоден для встраивания в runtime/log UI.

### 6. Tooltips, hints, onboarding microcopy

Подготовь source text для:

- кнопок подключения
- настройки языка
- `Сделать основным`
- TCP reachability
- technical log / human summary переключения

Лучше хранить это в i18n/help ресурсах, а не размазывать по коду.

## Что не нужно делать здесь

- рефакторинг runtime-кода
- сложная UI-механика
- build pipeline

## Проверки качества текста

Проверь каждый текст на три вопроса:

1. Поймёт ли это нетехнический пользователь?
2. Не теряется ли смысл для технического пользователя?
3. Не звучит ли это как дословный перевод интерфейса?

## Acceptance criteria

Работа считается готовой, если:

- `README.md` полностью переписан на русский
- встроенная справка подготовлена для `ru` и `en`
- welcome flow переписан человеческим языком
- есть словарь типовых ошибок
- английские тексты синхронны по смыслу с русскими

## Формат финального ответа агента

В конце сообщи:

1. Какие контентные файлы созданы или переписаны
2. Как организован help content
3. Какие текстовые решения были приняты по тону
4. Какие спорные термины пришлось объяснять отдельно
5. Что должен подхватить UI-поток для финальной встройки
